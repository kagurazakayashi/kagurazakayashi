### 最近更新的可用项目（过期项目会从此列表删除）

- 📔 自用代码备忘录 [CodeNotebook](https://github.com/kagurazakayashi/CodeNotebook) ，积累了很多年的个人知识库。
- 📋 供 Kiger 无声交流的实用 APP [长夜锦书](https://github.com/kagurazakayashi/EvernightBoard) 。
- 🐋 在 TrafficMonitor 中实时显示 DeepSeek 官网余额的插件 [TrafficMonitorPlugin-DeepSeekDeskBand](https://github.com/kagurazakayashi/TrafficMonitorPlugin-DeepSeekDeskBand) 。
- 🖼️ Flutter 不想传一大堆各平台各尺寸图标进仓库的话， [FlutterIconCreator](https://github.com/kagurazakayashi/FlutterIconCreator) 可以让你只需要传两张图片就够。
- 🐹 自用的 Golang 库 [libNyaruko_Go](https://github.com/kagurazakayashi/libNyaruko_Go) ，包含各种各样常见工具的封装。
- 🐹 让 Golang 程序像 Flutter 一样实施多语言，只需要执行 [go-gen-l10n](https://github.com/kagurazakayashi/go-gen-l10n) ，就可以生成静态多语言 go 代码文件。
- 🐹 一口气把 Golang 程序编译到所有平台或多个平台可以用 [go-build-all](https://github.com/MasaeProject/go-build-all)
- 🎨 [PixelDiff](https://github.com/kagurazakayashi/PixelDiff) 可以检测图片中指定坐标点的颜色值，并计算其与目标 RGB 颜色的量化差异度的 CLI 工具，通常同于游戏脚本中和 AI 自动化中进行图片元素判断。
- ⚛️ HTTP-to-NATS 网关桥接器 [ApiNatsBridge](https://github.com/MasaeProject/ApiNatsBridge) 将标准 HTTP REST 请求转换为 NATS 消息并转发至后端微服务，再将微服务的响应返回给 HTTP 客户端，还有防护和认证等扩展功能。
- 📑 将网页生成为多个版本的 [StaticDeployment](https://github.com/MasaeProject/StaticDeployment) 可以帮助你处理纯静态网站中的模板套用，支持插件。
- 📷 批量根据照片的实际拍摄时间和日期来重命名文件用的 CLI 工具 [长夜记忆](https://github.com/kagurazakayashi/EvernightMoments) ，也可以用于批处理脚本或 AI 自动化。
- 🖥️ 启动一个软件，当它运行结束后再运行另一个程序或命令的 Windows 工具 [NyarukoAppRelay](https://github.com/kagurazakayashi/NyarukoAppRelay) 可以让你串行地运行软件。
- 🖥️ 一个 Windows 用的超小 CLI 工具集 [NyarukoMiniTools](https://github.com/kagurazakayashi/NyarukoMiniTools) 实现了一些 Windows 平台下没有的命令，并且以小而快为目标，让写批处理文件更方便。
- 🖥️ [NyarukoWinUserMgr](https://github.com/kagurazakayashi/NyarukoWinUserMgr) 可以让你批量快速管理 Windows 用户账户。
- 🖼️ 通过 Windows 资源管理器右键菜单点击 [copy-image](https://github.com/kagurazakayashi/copy-image) 可以复制图片文件内容（而不是文件本身）的工具，支持限定尺寸。
- 📲 [iUImport](https://github.com/kagurazakayashi/iUImport) 可以让你在没有无线连接途径的情况下通过数据线将照片或视频轻松导入到 iOS 相册中。
- 🔖 [ExtractDefine](https://github.com/kagurazakayashi/ExtractDefine) 可以帮助你找到 C 程序中都定义了哪些宏，在其他程序和 C 程序组合编译脚本中会用到。
- 📑 VSCode 插件 [used-characters](https://github.com/kagurazakayashi/used-characters) 可以帮助您快速了解某段内容中都使用了哪些字符，通常在做字体精简等操作时会用到。
